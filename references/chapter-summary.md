# 章节总结.md 规范与模板

每章写完立即生成 `章节总结/第001章.md`。它不是复述全文，而是让未来的模型在不加载原文时恢复因果、状态和承诺。

## 文件模板

```yaml
chapter: 1
title: 章节名
word_count: 0
date_written: YYYY-MM-DD
pov: 视角人物
location: 地点
story_time: 故事内时间
previous_summary: 第000章
```

```markdown

用“因为A，主角选择B，导致C”写，不写“本章发生了很多事”。

## 开始状态
- 主角目标/情绪/知识/资源/伤势：
- 其他在场人物及目标：
- 地点、时间、规则限制：
- 上章遗留问题：

## 场景因果链
1. 场景目标：
   - 阻力：
   - 主角策略：
   - 结果与代价：
2. 场景目标：
   - 阻力：

## 本章不可逆变化
- 信息变化：谁知道了什么？
- 人物变化：目标、情绪、底线、关系如何变？
- 局势变化：资源、敌我、时间压力如何变？
- 能力/规则变化：新增、消耗、受限或被证明错误？

## 新增与确认事实
- F001：事实 + 来源场景

## 伏笔更新

## 结尾钩子

## 下一章接口
- 主角下一步最可能做什么：

- 本章目标是否完成或改变：是/否
```

## 压缩规则

将本章总结压缩成 80–150 字时，保留顺序：`主角目标 → 关键选择 → 代价 → 状态变化 → 新钩子`。不要压缩掉人物知道范围、未回收伏笔、资源消耗和时间地点。
