# 反馈与改稿.md 规范与模板

读者反馈是证据，不是命令。先判断反馈是否具体、是否重复出现、是否能在文本中找到证据，再决定局部修复还是保持原方向。

## 改稿卡

```markdown
### R001｜问题标题
- 证据段落/章节：
- 问题类型：主线/因果/人设/节奏/信息/视角/语言/作品包装
- 读者影响：困惑、无聊、不信、情绪断裂、追读动机下降
- 优先级：P0 结构致命 / P1 明显影响 / P2 润色
- 最小修复：
- 替代方案及代价：
- 受影响的伏笔、人物、时间线：
- 修改后验证：
```

## 改稿顺序

先修主线与因果，再修人物动机和关系，再修章节节奏、钩子和信息释放，最后修句子。一次大改前先复制项目快照，保留旧版本和变更原因；不要因为一条评论重写整卷。
