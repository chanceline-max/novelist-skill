# 小说质量检查清单

按任务选择检查项，不必每次全部执行。输出问题时标注“证据—影响—修改建议”。

## 选题与简介

- [ ] 一句话故事包含主角、目标、阻力和失败代价。
- [ ] 题材承诺与正文前几章一致。
- [ ] 卖点能转化为反复出现的事件，不只是宣传语。
- [ ] 主角身份或处境有明确的可写性。
- [ ] 长线冲突足以支撑预估篇幅。
- [ ] 设定新鲜点不会依赖复制现成作品。

## 开篇

- [ ] 读者能迅速判断主角是谁、正在面对什么。
- [ ] 前几页出现异常、目标、关系张力或明确问题。
- [ ] 必要设定通过事件和选择释放。
- [ ] 主角不是长时间旁观、听课或等待剧情开始。
- [ ] 开篇结尾形成新的问题或更高目标。
- [ ] 书名、简介、封面预期与开篇情绪相符。

## 人物

- [ ] 每个重要人物有独立目标和判断。
- [ ] 行为符合身份、成长环境和当前信息。
- [ ] 人物选择推动剧情，而不是被剧情拖走。
- [ ] 角色有可辨识的语言、细节或行为习惯。
- [ ] 反差有原因，不是为了惊讶而违背人设。
- [ ] 人物弧光由连续选择构成。
- [ ] 配角支线服务主线或主题，没有无效支线。

## 剧情与大纲

- [ ] 每个阶段都有目标、阻力、代价、局部结算。
- [ ] 前一事件能解释后一事件的发生。
- [ ] 冲突升级改变了风险、资源、关系或认知。
- [ ] 反转有前置线索，回看时能够成立。
- [ ] 副本/单元结束后改变主线或人物。
- [ ] 中段没有重复开局的循环。
- [ ] 结局兑现开篇承诺并回应核心主题。

## 单章与场景

- [ ] 本章有可说清的目标。
- [ ] 阻力具体，不是“大家都很紧张”。
- [ ] 主角采取了主动行动或做出选择。
- [ ] 中段发生了计划失效、信息变化或关系变化。
- [ ] 章节结束时状态与开始不同。
- [ ] 每个场景都服务主线、人物或主题至少一项。
- [ ] 过渡章制造了新的期待。
- [ ] 结尾钩子与本章因果相关。

## 语言与视角

- [ ] 叙述视角稳定，读者知道信息从谁的感知来。
- [ ] 对话有目的、差异和潜台词，不是轮流讲设定。
- [ ] 动作、反应和具体细节多于空泛形容。
- [ ] 删除重复的情绪、结论和背景说明。
- [ ] 句子长度和段落节奏匹配场景张力。
- [ ] 重要情绪通过行为与选择呈现。

## 连续性账本

- [ ] 时间线、年龄、地点和移动时间一致。
- [ ] 人物当前伤势、资源、关系和认知已更新。
- [ ] 世界规则前后一致，例外有解释和代价。
- [ ] 伏笔有记录，回收时没有凭空出现新信息。
- [ ] 章节承接上一章留下的问题或明确说明转场。
- [ ] 称谓、专有名词、能力等级和物品状态统一。

## 改稿决策

- [ ] 先定位问题，不在没有证据时整篇推倒。
- [ ] 区分局部修补、中段重排和主线重构。
- [ ] 修改前保存受影响伏笔、设定和后续依赖。
- [ ] 修改后重读相邻章节，检查衔接。
- [ ] 每次只验证少数高优先级假设。
- [ ] 修改后的文本仍保留作品的核心卖点和声音。

## 写作 skill 输出自检

- [ ] 是否先确认用户要策划、写正文、续写、改稿还是诊断？
- [ ] 是否复述了当前故事状态，而非凭空接写？
- [ ] 是否提供了具体可执行的下一步？
- [ ] 是否避免套话、万能大纲和无依据的流量承诺？
- [ ] 是否保留用户的风格、题材和限制？
- [ ] 是否在长任务中更新连续性账本？
