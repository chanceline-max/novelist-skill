# 小说写作执行框架

本文件把写作技巧语料转成模型可执行的决策框架。它是方法库，不是固定模板；根据题材、读者和作品阶段取用。

## 一、作品定位

### 一句话故事

使用：`一个【身份/处境】的主角，为了【可见目标】，必须在【核心阻力】下做出【关键行动】，否则【具体代价】。`

好的 premise 同时包含人物、目标、阻力和风险。只有“某人穿越后一路升级”还不是故事，因为缺少具体目标、阻力和代价。

### 卖点四类

1. **题材卖点**：读者熟悉的期待与一个明确的新鲜角度。
2. **人物卖点**：反差、关系张力、独特能力、身份秘密或不可替代的选择。
3. **机制卖点**：世界规则、任务系统、职业流程、推理规则或资源限制。
4. **情绪卖点**：爽、甜、虐、燃、悬疑、治愈、成长、复仇等可持续情绪。

卖点必须能转译为章节事件。不能只写在简介里，正文却无法反复兑现。

### 作品契约模板

```text
暂定名：
题材/子类型：
目标读者：
视角与人称：
一句话故事：
核心卖点（最多3项）：
读者开篇承诺：
长线主线：
预估篇幅与更新节奏：
尺度/禁区/必须保留的元素：
```

## 二、故事发动机

### 主角动力表

```text
外在目标：主角想得到什么？
内在需求：他真正需要学会或承认什么？
当前缺口：什么能力、关系、信息或资源不足？
核心恐惧：他最怕失去什么？
底线：什么事情即使付出代价也不做？
可被迫改变的条件：什么事件会逼他重新选择？
```

外在目标适合推动事件，内在需求适合支撑人物弧光。两者应有张力，例如主角想成为强者，却必须学会信任他人。

### 对手与阻力

对手不只是“更强的敌人”。为每个主要阻力写清：目标、资源、判断、底线、与主角的关系和失败后的反应。环境阻力包括制度、时间、距离、舆论、资源稀缺、规则限制和错误信息。

冲突升级优先顺序：

1. 让主角必须在两个重要目标间取舍。
2. 让原有策略失效，迫使主角换方法。
3. 提高失败代价或缩短时间窗口。
4. 揭露会改变判断的新信息。
5. 让成功也产生新的责任、关系或风险。

### 循环任务

长篇需要可重复但不重复的阶段循环：接到目标 → 调查/准备 → 遇到规则或阻力 → 付出代价 → 局部胜利/失败 → 得到新信息 → 进入更高难度。

每一轮至少改变一项：目标、规则、关系、资源、认知或代价。若六项都不变，就是流水账。

## 三、长篇结构

### 七节点总纲

1. **开端异常**：日常被打破，主角被迫应对。
2. **第一次主动选择**：主角不再只是被动逃命，明确追求一个目标。
3. **第一次大代价**：展示规则和失败后果，打破轻松幻想。
4. **中点重定义**：新信息改变主角对目标、敌人或自我的理解。
5. **最低谷**：原策略失败，重要关系或资源受损。
6. **最终选择**：主角用新的认知和代价做出不可撤回的选择。
7. **余波兑现**：结局回应开篇承诺，同时展示选择造成的后果。

### 分卷设计

```text
卷名/主题：
本卷问题：
本卷目标：
入口状态：
中点变化：
本卷主要对手：
关键副本/单元：
本卷付出的代价：
卷尾结算：
下一卷钩子：
```

每卷都要有局部闭环。读者应感觉本卷的问题被回答了一部分，同时更大的问题被打开。

### 副本/单元设计

```text
入口：为什么主角必须进入？
独立目标：完成什么才算通过？
规则：什么可做、不可做、会付出什么代价？
主要角色：谁与主角合作或竞争？
三次升级：阻力如何逐步变难？
关键选择：主角必须放弃或承担什么？
结算：奖励、损失、信息和关系变化是什么？
主线影响：它如何改变后续故事？
```

副本必须改变主角或主线，不能只是展示世界观的旅游景点。

## 四、人物塑造

### 人物卡

```text
姓名/身份：
表面目标：
真实需求：
过去事件与伤口：
核心信念：
秘密：
底线与例外：
能力/短板：
关系需求：
行为反差：
语言节奏/口头习惯：
关键选择：
弧光终点：
```

人物不要靠作者评价来“宣布很聪明/很善良”。让读者从选择、反应、细节、对话和付出的代价中推断人物。

### 让人物活起来

- 先设定人物的追求，再设计阻止追求的事件。
- 让同一事件在不同人物眼里有不同意义。
- 让配角拥有自己的目标和判断，不只围着主角赞美或送工具。
- 用生活细节、职业习惯、身体反应和说话方式强化记忆点。
- 用反差增加立体感，但反差必须能解释，不能为了反差而违背人设。
- 让人物在冲突中做选择；连续选择比单次背景介绍更能形成成长。

## 五、场景、章节和节奏

### 场景卡

```text
场景目的：没有这一场，主线会缺什么？
进入状态：人物带着什么目标和情绪进场？
阻力：谁/什么让目标无法直接完成？
策略：人物如何试图解决？
转折：什么信息或行动改变局面？
退出状态：人物得到/失去什么？
承接：下一场被什么因果推动？
```

### 单章任务

章节不是“写满字数”，而是完成一个可感知的变化。用以下格式规划：

```text
本章承接：上一章留下的目标/问题
本章目标：主角本章要完成什么
主要阻力：为什么不能轻易完成
关键动作：主角主动做什么
中段转折：原计划如何失效
本章结果：成功、失败或部分成功
新增信息/伏笔：
结尾钩子：读者为什么继续
```

节奏问题通常不是“每章都要爆点”，而是目标、阻力、结果和期待长期没有变化。过渡章也要制造新期待：关系变化、信息差、时间限制、危险逼近或下一目标的入口。

### 开篇检查

前几页尽快让读者看到：主角处境、当前异常、可理解的目标、独特卖点或关系张力。删掉不影响选择的背景说明，把必要设定放进正在发生的事件里。

### 信息释放

每段说明都问：读者现在必须知道吗？能否改成行动、对话、误解、选择或后果？不要一次解释所有规则；让规则在人物付出代价时被理解。

## 六、改稿与数据反馈

### 改稿顺序

先改影响最大的结构问题，再改章节节奏，再改人物和语言。推荐顺序：

1. 主线目标是否清楚。
2. 冲突是否真实且逐步升级。
3. 章节是否有变化和结尾期待。
4. 人物行为是否符合动机与信息。
5. 视角、时间线和设定是否一致。
6. 重复、冗余、空泛形容和句子问题。

### 越改越乱的防护

大改前建立依赖清单：伏笔、规则、人物状态、关系进度、时间线、后续章节已使用的信息。先区分小毒点、局部问题和大剧情问题。无法驾驭后续衔接时不要贸然改动主线；若必须大改，应先重建受影响章节的骨架。

### 数据转译

作品数据只能作为信号：

- 展示到阅读少：检查书名、封面、简介和开篇承诺是否一致。
- 开篇跳出多：检查进入冲突速度、信息理解成本和主角目标。
- 追读下降：检查章节结尾、阶段目标和主线推进。
- 完读下降：检查单章目标、冲突兑现、冗余和视角切换。
- 评论集中质疑：区分读者偏好与逻辑/人设/信息错误，不要盲目迎合单一评论。

数据不能替代作者判断；修改后重读并比较前后效果。
