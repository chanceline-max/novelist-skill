# 长篇小说项目文件系统

长篇创作不能只依赖聊天记录。每次创作前读取摘要和当前状态，每次创作后更新账本，才能在上下文截断、换模型或隔天续写时恢复同一部作品。

## 必备文件

```text
小说项目/
├─ 大纲.md                 # 主线、分卷、阶段目标与结局
├─ 伏笔.md                 # 所有承诺、悬念、线索和回收状态
├─ 人物.md                 # 主角、配角、反派、关系状态
├─ 世界观.md               # 规则、等级、资源、地理、历史
├─ 时间线.md               # 故事内时间与事件顺序
├─ 章节索引.md             # 每章一句话、视角、状态变化、钩子
├─ 章节总结/第000章.md     # 每章的可压缩上下文
├─ 素材库.md               # 可用场景、职业细节、意象、资料来源
└─ 反馈与改稿.md           # 读者反馈、问题证据、修复决策
```

## 初始化顺序

1. 先写 `大纲.md` 的作品契约、主线句和结局方向。
2. 根据大纲建立 `人物.md`、`世界观.md`、`时间线.md`。
3. 预埋伏笔时立即登记到 `伏笔.md`，不要靠记忆。
4. 写完一章后依次更新：章节总结、章节索引、人物状态、时间线、伏笔状态；若规则或资源变化，再更新世界观。

## 读取顺序

续写前只需先读取：最近 3 章总结、`章节索引.md` 的最近 10 章、`大纲.md` 当前卷、`伏笔.md` 未回收项、人物和世界观中受本章影响的部分。遇到冲突再回溯原文，不要默认全文加载。

## 文件写作规则

- 每条事实标注来源章节；推断用 `INFERRED`，未确认用 `UNKNOWN`。
- 不删除旧记录；修改用“旧值 → 新值 + 原因 + 影响章节”记录。
- 每个文件顶部保留“最后更新章节、最后更新时间、待核对项”。
- 文件只记录会影响后续写作的事实，不把散文式感想混进账本。
- 任何新设定都要同时登记规则、限制、首次出现章节和潜在后果。

## 上下文压缩等级

- **L0 原文**：完整章节，仅在局部改稿或精确引用时读取。
- **L1 章节总结**：本章 300–800 字，记录因果、事实、状态和钩子。
- **L2 阶段摘要**：每 5–10 章压缩成 500–1000 字，记录阶段变化和未决线。
- **L3 项目快照**：当前目标、主要人物状态、最近冲突、三条未决问题，控制在 200 字内。

当上下文接近上限时，先生成 L1，再把多个 L1 合并成 L2，最后只携带 L3 与当前章卡继续写。压缩不得丢失“谁知道什么、谁欠谁什么、哪些伏笔尚未回收”。
