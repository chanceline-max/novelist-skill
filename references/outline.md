# 大纲.md 规范与模板

`大纲.md` 是全书的导航，不是不可修改的合同。它必须足够具体地防止迷路，又保留创作时发现新路径的空间。

## 文件头

```yaml
title: 暂定书名
version: 0.1
last_updated_chapter: 0
last_updated_at: YYYY-MM-DD
status: planning|serializing|completed|paused
certainty: confirmed|draft
```

## 一、作品契约

```markdown
## 作品契约
- 题材/子类型：
- 目标读者：
- 核心情绪：
- 一句话卖点：熟悉题材 + 反常机制 + 主角目标 + 失败代价
- 读者承诺：前3章得到什么；中段持续期待什么；结局兑现什么
- 视角/时态/叙事距离：
- 预计总字数、卷数、单章字数、更新节奏：
- 明确不写的内容和逻辑边界：
```

## 二、主线句与故事发动机

使用：`谁 + 在什么困境下 + 为达成什么目标 + 对抗什么力量 + 最终获得/失去什么`。

```markdown
主线句：
主角外在目标：
主角内在缺口/错误信念：
对手目标：
对手可执行资源：
主角初始资源与限制：
不行动的代价：
每一轮循环：欲望 -> 阻力 -> 选择 -> 代价 -> 新问题
最终选择：
结局兑现的开篇承诺：
```

如果主线句无法让人预测前三章的行动，先重写主线句，不要继续扩充世界观。

## 三、总纲七节点

```markdown
1. 开端异常/诱因：
2. 第一转折（主角无法回到原状）：
3. 第一阶段目标：
4. 中点反转（信息、关系或目标改变）：
5. 最低谷（不行动或失败的代价最大化）：
6. 最终决战/最终选择：
7. 结局余波与下一部/下一卷接口：
```

每个节点写“触发事件、主角选择、付出代价、状态变化、读者新增问题”，禁止只写“发生大战”“关系升温”。

## 四、分卷表

| 卷 | 核心目标 | 主要阻力 | 核心变化 | 黄金节奏点 | 结算代价 | 卷尾钩子 |
|---|---|---|---|---|---|---|
| 1 |  |  |  |  |  |  |

一卷必须完成一个阶段问题，同时改变主角能力、关系、认知或代价。卷尾钩子应由本卷因果产生，不得凭空出现更大的敌人。

## 五、单元/副本卡

```markdown
### 单元：
- 入口条件：
- 单元问题：
- 主角阶段目标：
- 对手/环境的主动计划：
- 规则与隐藏条件：
- 资源限制与失败代价：
- 3个阶段性小目标：
- 中点反转：
- 结算奖励与新债务：
- 对主线/人物/关系的影响：
```

## 六、章节表

| 章 | 视角 | 开始状态 | 本章目标 | 阻力动作 | 关键选择 | 代价 | 状态变化 | 新信息 | 结尾钩子 | 需更新文件 |
|---|---|---|---|---|---|---|---|---|---|---|

每次只细化未来 1–3 章。每 4000 字至少拆出两个有结果的小情节；每个小情节必须改变局势、关系、信息或资源。
