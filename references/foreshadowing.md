# 伏笔.md 规范与回收系统

伏笔不是“埋一个神秘东西”，而是对读者做出未来会解释、兑现或反转的承诺。所有伏笔都登记，不用记忆管理。

## 伏笔表

| ID | 类型 | 埋设内容 | 首次章节 | 读者可见度 | 真正答案/回收动作 | 计划回收 | 当前状态 | 关联人物/规则 | 失误风险 |
|---|---|---|---:|---|---|---:|---|---|---|
| P001 | 身份/规则/物件/关系/悬念 |  |  | 明示/暗示/隐藏 |  |  | planted |  |  |

状态只能使用：`idea`（想法）、`planted`（已埋）、`hinted`（已加强）、`misdirected`（已误导）、`paid`（已回收）、`invalidated`（明确取消）、`overdue`（超期未回收）。

## 每个伏笔的完整卡片

```markdown
### P001｜名称
- 一句话承诺：读者被暗示未来会知道什么？
- 埋设事实：当前文本实际出现了什么？
- 首次出现章节/场景：
- 读者知道的层级：主角知道 / 读者知道 / 双方都不知道
- 真相与替代解释：
- 最少需要几次提示：
- 计划回收区间：
- 回收方式：信息揭示 / 行动后果 / 物件使用 / 关系选择 / 反转
- 回收时改变什么：主线、人物、规则、资源或情绪
- 当前状态：
- 如果删除会影响什么：
```

## 埋设规则

1. 伏笔必须能改变未来选择、重新解释过去，或提升某次回收的情绪价值；只制造神秘感而不影响故事的线索应删除。
2. 重大真相至少提前给出两类可回溯证据：一个事实线索和一个行为/情绪线索。
3. 回收前检查读者是否有公平机会推理；回收后补记“哪些章节被重新解释”。
4. 不能为了制造反转临时改写世界规则；如确需改规则，记录前置铺垫和付出的代价。
5. 每 5–10 章检查一次过期伏笔；超期时选择回收、延后并补提示、合并，或明确取消。

## 伏笔审计

写新章前列出本章可触发的 3 个未回收项，判断是否自然；写完后标记新增、加强、误导、回收和失效项。季度/大卷审计时检查：埋设与答案是否对应、提示是否过密、回收是否改变状态、读者是否会觉得答案凭空出现。
