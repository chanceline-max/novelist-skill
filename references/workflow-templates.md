# 可执行工作流与输出模板

本文件把“写小说”拆成可反复调用的协议。模型每次只推进一个阶段；如果关键输入缺失，先用默认值并标注假设，不要把不确定内容伪装成既定事实。

## 0. 统一工作协议

每次响应都先判断请求属于：策划、开篇、章节、续写、改稿、诊断、卡文、连载管理。然后遵循：

1. **读取上下文**：作品契约、最近章节、当前卷目标、连续性账本。
2. **声明假设**：列出最多 5 个会影响结果的未知项及采用的默认值。
3. **先给决策，后给正文**：先说明本轮要解决的矛盾与选择，再输出大纲/样稿。
4. **设置阶段闸门**：未确认核心冲突、人物动机、章节目标时，不直接写长篇正文。
5. **交付可复用产物**：每轮至少留下一个可复制的卡片、表格、清单或账本更新。

## 1. 新书接入问卷

最少询问 6 项：题材/子类型、目标读者、主角、核心欲望、主要阻力、希望本轮产物。可选询问：篇幅、更新频率、视角、尺度边界、参考作品（只分析方法，不仿写）。

输出“作品契约”：

```yaml
title: 暂定名
genre: 题材与子类型
audience: 读者在意的情绪/爽点/关系
promise: 一句话承诺
protagonist: 主角与初始处境
engine: 欲望 -> 阻力 -> 选择 -> 代价 -> 新问题
selling_points: [差异化卖点1, 卖点2, 卖点3]
tone_pov: 语气、视角、时态
length_rhythm: 预计篇幅与单章字数
boundaries: 不写内容、逻辑禁区
unknowns: 尚未确定的事项
```

若用户只有一句模糊想法，先给 3 个差异明显的方向；每个方向都写“读者承诺、发动机、前三章风险、适合的连载节奏”，再让用户选一个。

## 2. 选题与卖点协议

对每个候选题打 1–5 分：新鲜度、可连续产出、主角主动性、冲突密度、读者情绪回报、作者熟悉度。总分不是结论，必须解释最低分项如何修复。

卖点必须写成“对象 + 机制 + 情绪回报”，例如“失去记忆的鉴证师，通过每件遗物看到凶手最后一分钟，逐案拼回自己的身份”。禁止只写“很爽、很虐、很反转”。

## 3. 大纲生成协议

按“总纲 -> 分卷 -> 单元/副本 -> 章节 -> 场景”收敛。每层都回答：目标、阻力、关键选择、代价、状态变化、下一层钩子。

章节表格固定列：`章号 | 视角 | 本章目标 | 阻力 | 关键行动 | 新信息 | 关系/能力变化 | 代价 | 结尾钩子 | 账本更新`。

未通过以下闸门不得开始连续写正文：主线目标可用一句话复述；主角有当下必须行动的理由；反派/环境能主动制造阻力；本章结束状态与开始状态不同；结尾问题不是靠巧合解决。

## 4. 人物与关系协议

核心人物卡至少包含：表层目标、深层需求、恐惧、秘密、价值底线、错误信念、资源、弱点、说话节奏、关键关系、三次被迫选择、结局代价。每次人物行动先检查“他知道什么、想得到什么、最怕失去什么”。

关系线用状态记录：`关系初值 -> 触发事件 -> 误解/利益 -> 选择 -> 新状态`。感情推进必须由共同经历和不可逆选择产生，不能只靠告白或旁白宣布。

## 5. 章节与场景生成算法

写正文前先输出 5 行章卡：

```text
本章任务：
开场未稳定状态：
主要阻力及其主动动作：
主角的选择与代价：
结尾新问题/钩子：
```

然后将章节拆为 3–7 个场景。每个场景执行“目标 -> 阻力升级 -> 策略 -> 结果 -> 新问题”；至少一次让策略失败、信息改变或关系改变。开头尽量从正在发生的异常/冲突切入，中段用行动释放设定，结尾留下具体且有因果的未决事项。

写作输出默认分两段：先给 100–300 字试写或场景样稿，确认语气后再扩写；用户明确要整章时，仍先列章卡与承接检查。

## 6. 续写协议

先复述最近章节的：地点、时间、在场人物、已知事实、未决问题、人物当前情绪和资源。生成 2–3 条走向（稳妥推进/关系推进/风险升级），说明每条的短期收益与后续成本，选定后再写。续写结束必须输出：新增事实、人物状态变化、伏笔状态、下一章可接入口。

## 7. 改稿与诊断协议

按优先级排查：

1. 读者是否知道主角要什么；
2. 冲突是否具体且有代价；
3. 因果链是否断裂或靠巧合；
4. 人物行动是否违背动机；
5. 本章是否发生可感知变化；
6. 信息是否重复、过早解释或遗漏；
7. 句子、节奏、修辞是否影响阅读。

每个问题用“证据 -> 影响 -> 严重级别(P0/P1/P2) -> 最小修复 -> 验证方式”输出。先修 P0（主线/逻辑/人设），再修 P1（节奏/信息/情绪），最后修 P2（语言）。不要把全文改成同一种腔调。

## 8. 卡文决策树

- 目标不清：给主角一个本章必须完成且可失败的任务。
- 阻力太弱：加入有能力、有理由、会行动的反对者，并明确代价。
- 人物不动：改变其掌握的信息或提高“不行动”的损失。
- 中段重复：引入新规则/新关系/新敌人，或重新定义阶段目标。
- 设定过载：把设定转为会影响选择的事件，删除不能改变决策的说明。
- 大纲失控：暂停写正文，重建当前状态表与未来三章的最短路径。

每次卡文至少生成三条路线：主线推进、关系/情绪推进、风险/反转推进；说明各自会牺牲什么。

## 9. 连载交付协议

每章完成后更新连续性账本；维护“已解决伏笔、未解决伏笔、承诺回收期限、角色状态、时间地点、规则与资源”。数据反馈只能作为假设，不可单凭短期点击推翻作品核心契约。
